You must extend the existing cognitive assessment application with advanced analytical and interpretive modules. The goal is to transform the system from a simple task-based evaluator into a structured neurocognitive profiling tool with clinically interpretable outputs and parent-friendly feedback.

Do NOT remove or alter previously implemented functionality. All new modules must integrate with the current system architecture.

---

1. AUTOMATED NEUROCOGNITIVE PROFILE GENERATION

---

After completion of the cognitive tasks, the system must automatically generate a structured neurocognitive profile of the child based on task performance.

The profile must analyze and summarize performance across domains including:

* phonological working memory
* visuospatial working memory
* executive control
* language processing
* attention regulation
* response inhibition
* processing speed

The system must analyze:

• accuracy
• reaction times
• error types
• task persistence
• fatigue indicators

Output must include:

* visual performance indicators (graphs or radar chart)
* domain-based interpretation
* relative strengths and weaknesses

The goal is not to provide a medical diagnosis but a **functional cognitive profile**.

---

2. AALISS PROFILE DETECTION MODULE

---

Implement automated detection of the **AALISS cognitive performance profile** based on behavioral and performance patterns across tasks.

The system must analyze patterns including:

* Attention variability
* Articulation or phonological response errors
* Language processing efficiency
* Inhibitory control performance
* Sequencing accuracy
* Strategy use patterns

Based on these indicators, the system should produce an interpretive profile describing how the child tends to process information.

The system must present results as:

• descriptive categories
• pattern tendencies
• areas of strength
• areas requiring support.

The system must explicitly state that this profile represents **behavioral indicators and not a clinical diagnosis**.

---

3. ERROR PATTERN DETECTION SYSTEM

---

The application must classify errors automatically during task performance.

Error categories should include:

PHONOLOGICAL ERRORS
Examples:

* sound substitutions
* syllable omissions
* phoneme order errors.

EXECUTIVE CONTROL ERRORS

Examples:

* rule violation
* premature responses
* distractor responses
* switching failure.

MEMORY ERRORS

Examples:

* sequence truncation
* order inversion
* recall omissions.

SEMANTIC ERRORS

Examples:

* category intrusions
* meaning-based substitutions.

The system should generate a **pattern analysis summary**, identifying which error types occur most frequently.

---

4. CHILD VOICE RECOGNITION CONTROL

---

Implement strict voice recognition controls to ensure the system only captures the child’s intended response.

Requirements:

* Speech recognition must activate only during a predefined response window (e.g., 3–5 seconds).
* Microphone input must automatically deactivate after the response window ends.
* Background audio filtering must minimize external voices or noise.
* Implement voice activity detection to capture only the primary speaking voice.
* Prevent continuous recording or unintended transcription.

Optional enhancement:

* Allow the professional to manually repeat the trial if recognition fails.

---

5. SCORE INTERPRETATION SYSTEM

---

The system must translate numerical task scores into interpretable cognitive indicators.

Each domain must include:

* score value
* percentile or relative performance indicator
* interpretation label (e.g., strong, typical, developing, or needs support)

The report must clearly explain what each score represents in functional terms, such as:

* ability to hold verbal information temporarily
* ability to control attention
* ability to manage multiple task demands.

Avoid deterministic language or diagnostic claims.

---

6. PARENT-FRIENDLY PSYCHOEDUCATION SECTION

---

At the end of the report, include a separate section written in accessible language for parents or guardians.

This section must:

* explain what working memory and attention are
* describe how these processes influence learning
* highlight the child’s strengths
* provide simple strategies to support cognitive development at home.

Examples of recommendations:

* breaking instructions into smaller steps
* using visual reminders
* practicing memory games
* reducing distraction during tasks.

Language must be supportive and non-alarmist.

---

7. CLINICAL REFERRAL GUIDANCE

---

If the system detects consistently low performance across multiple cognitive domains or significant error patterns, it should generate a **professional recommendation section**.

This section must:

* suggest that further evaluation may be beneficial
* recommend consultation with qualified professionals such as:

• child psychologist
• neuropsychologist
• speech-language therapist
• educational specialist.

The system must emphasize that the application does NOT provide diagnoses.

---

8. MULTI-LAYER REPORT STRUCTURE

---

The final report must contain three sections:

SECTION 1 — Professional Cognitive Summary
Detailed cognitive analysis with metrics and graphs.

SECTION 2 — Neurocognitive Pattern Interpretation
AALISS profile + error pattern analysis.

SECTION 3 — Parent Guidance and Psychoeducation
Friendly explanation and home strategies.

---

9. SAFETY AND ETHICAL CONSTRAINTS

---

The system must include clear statements that:

* the assessment is a screening tool
* results should be interpreted by trained professionals
* results are not diagnostic conclusions.
